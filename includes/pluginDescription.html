Skill Inspector

Skill Inspector brings IDE-grade feedback to Agent Skill authoring. It helps you write, review, and maintain SKILL.md files with the same kind of immediate validation you expect when working with code.

Agent Skills are becoming a common way to package reusable instructions, scripts, references, and assets for AI coding agents. A small mistake in the frontmatter, a broken local reference, or an overly broad tool permission can make a skill harder to discover, harder to run, or riskier to share. Skill Inspector focuses on those authoring-time problems directly inside JetBrains IDEs.

What It Checks

The plugin uses the public Agent Skills specification as its default baseline, then adds practical authoring checks for the files and folders around a skill.

  • Skill structure: validates YAML frontmatter, required fields, naming rules, and the relationship between the skill directory and frontmatter name.
  • Authoring quality: highlights missing usage guidance, missing document structure, and skill files that are becoming too large for a lightweight entry point.
  • Local references: checks relative Markdown links so referenced files, scripts, and assets do not silently break during editing or migration.
  • Safety signals: flags suspicious secrets, dangerous command snippets, overly broad tool permissions, sensitive local paths, and prompt-injection style text.
  • Conservative Quick Fixes: fixes deterministic structure issues such as missing frontmatter, missing required fields, and mismatched skill names.

Where It Helps

  • Creating a new Agent Skill from scratch.
  • Reviewing a team skill before committing or publishing it.
  • Migrating skills between Claude Code, Codex, Cursor, Junie, Copilot, or other Agent Skill consumers.
  • Keeping local reference files, scripts, and assets connected to the main SKILL.md.

Design Approach

  • Specification-first: generic Agent Skills rules are treated as the default baseline instead of one vendor-specific dialect.
  • Low surprise: warnings are intended to explain concrete authoring risks, not to enforce a single writing style.
  • Conservative fixes: Quick Fixes repair structure but do not rewrite your instructions.
  • Local-first: this version focuses on local IDE inspection, not remote registries or skill synchronization.

检查内容

插件以公开的 Agent Skills specification 作为默认基准，并结合 skill 周边文件和目录结构做实际编写场景中的检查。

  • Skill 结构：校验 YAML frontmatter、必填字段、命名规则，以及 skill 目录名和 frontmatter name 的一致性。
  • 编写质量：提示缺少使用说明、缺少文档结构，以及主 SKILL.md 过长等问题。
  • 本地引用：检查相对 Markdown 链接，避免参考文件、脚本和资源在编辑或迁移过程中悄悄失效。
  • 安全信号：提示疑似密钥、危险命令片段、过宽工具权限、敏感本地路径和 prompt injection 风格文本。
  • 保守 Quick Fix：只修复确定性的结构问题，例如缺少 frontmatter、缺少必填字段、skill 名称不一致。

适用场景

  • 从零创建新的 Agent Skill。
  • 在提交或发布前审查团队维护的 skill。
  • 在 Claude Code、Codex、Cursor、Junie、Copilot 或其他 Agent Skill 消费方之间迁移 skill。
  • 维护 SKILL.md 与本地参考文件、脚本、资源文件之间的连接关系。

设计取向

  • 规范优先：以通用 Agent Skills 规则作为默认基准，而不是绑定某个厂商的私有方言。
  • 低打扰：提示应说明具体编写风险，而不是强迫所有 skill 使用同一种写作风格。
  • 保守修复：Quick Fix 修复结构，不自动改写用户写下的指令内容。
  • 本地优先：当前版本聚焦 IDE 内本地检查，不包含远端 registry 或 skill 同步能力。
