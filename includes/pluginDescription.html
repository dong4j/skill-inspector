Skill Inspector

Skill Inspector is a JetBrains IDE plugin for writing and reviewing Agent Skills. It brings real-time inspection, problem navigation, and conservative Quick Fixes to SKILL.md files, so skill authors can catch structure, reference, quality, and safety issues while editing.

Agent Skills package reusable instructions, scripts, references, and assets for AI coding agents. A broken frontmatter block, a missing local reference, or an overly broad tool permission can make a skill harder to discover, harder to run, or unsafe to share. Skill Inspector focuses on those authoring-time problems directly inside JetBrains IDEs.

Core Capabilities

  • Specification-first checks: validates SKILL.md frontmatter, required fields, name format, description limits, and the relationship between the skill directory name and frontmatter name.
  • Authoring quality hints: highlights missing usage guidance, missing document structure, generic descriptions, and overly long skill entry files.
  • Reference and resource validation: checks Markdown links, missing files, paths outside the skill directory, path casing mismatches, unused references/ files, and scripts that are not explained in SKILL.md.
  • Safety signals: flags suspicious secrets, dangerous shell snippets, broad allowed-tools: Bash permissions, sensitive local paths, and prompt-injection style text.
  • Conservative Quick Fixes: repairs deterministic structure issues such as missing frontmatter, missing required fields, invalid skill names, and missing local reference files.
  • IDE workflow integration: reports issues in Problems, supports right-click validation for project skills, provides a current file validation button in the editor, and shows the current file Error / Warning count in the status bar.

How To Use

  • Open a SKILL.md file and inspect the highlighted problems directly in the editor or Problems tool window.
  • Use available Quick Fixes for deterministic repairs such as adding frontmatter, required fields, or missing reference files.
  • Use the editor right-click action to validate all SKILL.md files in the current project.
  • Use the bottom-right editor validation button when you want to re-check the current skill explicitly.
  • Use Settings | Tools | Skill Inspector to enable or disable the plugin's SKILL.md inspection.

Design Approach

  • Specification-first: generic Agent Skills rules are treated as the default baseline instead of one vendor-specific dialect.
  • Low surprise: warnings explain concrete authoring risks without enforcing a single writing style.
  • Conservative fixes: Quick Fixes repair structure but do not rewrite your instructions or silently remove content.
  • Local-first: inspection runs against local project files and keeps feedback in the IDE workflow.

Skill Inspector

Skill Inspector 是一个面向 JetBrains IDE 的 Agent Skill 编写与审查插件。它为 SKILL.md 提供实时 Inspection、Problems 定位和保守 Quick Fix，让 skill 作者在编辑时就能发现结构、引用、质量和安全问题。

Agent Skills 用于把可复用的指令、脚本、参考文档和资源文件打包给 AI coding agent 使用。frontmatter 错误、本地引用失效、工具权限过宽，都会让 skill 更难被发现、更难执行，甚至不适合分享。Skill Inspector 专注解决这些发生在编写阶段的问题，并把反馈直接放在 JetBrains IDE 里。

核心能力

  • 规范优先检查：校验 SKILL.md frontmatter、必填字段、名称格式、描述长度，以及 skill 目录名和 frontmatter name 的一致性。
  • 编写质量提示：提示缺少使用时机、缺少文档结构、描述过泛、主 SKILL.md 过长等问题。
  • 引用与资源校验：检查 Markdown 链接、缺失文件、越界路径、大小写不一致、未引用的 references/ 文件，以及未在 SKILL.md 中说明用途的脚本。
  • 安全信号：提示疑似密钥、危险 shell 片段、过宽的 allowed-tools: Bash 权限、敏感本地路径和 prompt injection 风格文本。
  • 保守 Quick Fix：只修复确定性的结构问题，例如缺少 frontmatter、缺少必填字段、非法 skill 名称、缺失本地引用文件。
  • IDE 工作流集成：在 Problems 中展示问题，支持右键校验项目内 skill、编辑器当前文件校验按钮，并在状态栏展示当前文件 Error / Warning 数。

如何使用

  • 打开 SKILL.md，直接在编辑器或 Problems 工具窗口查看高亮问题。
  • 对确定性结构问题使用 Quick Fix，例如补 frontmatter、补必填字段、创建缺失引用文件。
  • 使用编辑器右键菜单校验当前项目中的所有 SKILL.md。
  • 需要显式重检当前 skill 时，使用编辑器右下角校验按钮。
  • 通过 Settings | Tools | Skill Inspector 启用或禁用 SKILL.md Inspection。

设计取向

  • 规范优先：以通用 Agent Skills 规则作为默认基准，而不是绑定某个厂商的私有方言。
  • 低打扰：提示说明具体编写风险，不强迫所有 skill 使用同一种写作风格。
  • 保守修复：Quick Fix 修复结构，不自动改写指令正文，也不静默删除内容。
  • 本地优先：检查面向本地项目文件，并把反馈保留在 IDE 工作流中。
